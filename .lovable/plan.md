

## Исправление фото товаров и расширение карточки товара

### Проблема 1: Неподходящие фото

Часть товаров получила случайные картинки с Unsplash (подсолнухи, пейзажи, фото из лаборатории, кактус) вместо тюльпанов, а некоторые карточки пустые. Причина — нестабильные `?w=1200&q=80` ссылки и неточно подобранные фото-ID.

**Решение:** заменю изображения у всех 15 товаров на проверенные локальные ассеты, которые уже есть в проекте (`bouquet-pink`, `bouquet-white`, `bouquet-yellow`, `bouquet-mix`, `bouquet-peony`, `by-stem-25`, `by-stem-51`). Они подгружаются через `resolveImage` и точно соответствуют цвету и типу букета. Дополнительно подберу 1–2 надёжные Unsplash-ссылки именно с тюльпанами как вторичные фото для галереи (с прямой проверкой, что URL отдаёт тюльпан).

Обновление сделаю SQL-запросом UPDATE по `id` каждого товара — без миграции, через insert-tool.

### Проблема 2: На странице товара нет описания и вариантов

Сейчас `/catalog/$id` показывает только: название, цену, состав, цвет, кол-во. Нет:
- развёрнутого описания (повод, сорт, стойкость, уход)
- выбора размера/комплектации (например, 15 / 25 / 51 стебель)
- выбора упаковки (крафт / лента / тишью)
- выбора открытки

**Решение:**

**A. Расширю описания** в БД — для каждого из 15 товаров пропишу подробный текст (3–5 предложений): про сорт, длину стебля, упаковку, повод, как ухаживать, сколько простоит.

**B. Добавлю в карточку товара блок «Варианты»** прямо в `src/routes/catalog.$id.tsx`. Поскольку схема `products` уже содержит `stems_count` и `price`, варианты количества буду генерировать на лету:

- **Количество стеблей** (для букетов): 15 / 25 / 51 / 75 — кнопки-чипы. При смене пересчитывается итоговая цена пропорционально базовой (`price / stems_count * выбранное`).
- **Упаковка**: Крафт-бумага / Атласная лента / Прозрачная плёнка — чипы, +0 / +300 / +200 ₽ к цене.
- **Открытка**: Без открытки / С открыткой (+150 ₽) — toggle, текст пожелания во `textarea`.

Выбранные параметры:
- отображаются в итоговой цене сверху кнопки «В корзину»
- сохраняются в позиции корзины (имя товара дополняется: «Розовый рассвет · 51 шт · с лентой»)
- влияют на итоговую цену в корзине

**C. Доп. секции на странице товара:**
- «Что входит» — список из `composition` + упаковка + открытка
- «Уход и стойкость» — статичный блок с иконками (4 пункта: подрезать стебли, менять воду, держать в прохладе, простоит 5–7 дней)
- Галерея миниатюр под основным фото (если в `images` несколько URL)

### Технические детали

**Файлы для изменения:**
- `src/routes/catalog.$id.tsx` — добавить состояния `selectedStems`, `selectedPackaging`, `withCard`, `cardText`; UI блока вариантов; пересчёт цены; передача расширенного `name` в `cart.add`.
- `src/lib/products.ts` — добавить хелперы `packagingOptions`, `cardPriceDelta`, `computeVariantPrice`.
- БД: один UPDATE-запрос на каждый товар с новыми `images` (локальные ключи) и расширенным `description`.

**Порядок выполнения:**
1. Обновить данные товаров (картинки + описания) через insert tool.
2. Расширить `src/lib/products.ts` константами вариантов.
3. Переписать страницу товара с блоком вариантов и галереей.
4. Проверить, что цена в корзине считается из переданного значения.

### Что не меняем

- Схему БД (поля `images`, `description` уже есть).
- Каталог `/catalog` и `ProductCard` — там по-прежнему базовая цена.
- Админку — она уже умеет редактировать `description` и `images`.
