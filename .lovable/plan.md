## Текущее состояние

Админка по адресу `/admin` уже умеет почти всё, что вы просите:

**Заказы (`/admin/orders`):**
- Список всех заявок с датой, клиентом, телефоном, суммой, статусом
- Фильтры по статусу: Все / Новая / В работе / Выполнена / Отменена
- Боковая панель с деталями: адрес, дата и время доставки, комментарий, состав
- Смена статуса в один клик
- Поле «Заметки администратора»

**Товары (`/admin/products`):**
- Сетка карточек со всеми товарами
- Создание / редактирование / удаление
- Поля: название, описание, состав, цена, тип, цвет, кол-во стеблей
- Загрузка фото с компьютера в bucket `product-images` (уже работает)
- Чекбоксы «В наличии» и «Показывать на сайте»
- Удаление загруженных фото

**Дашборд (`/admin`):**
- Новые заявки, всего товаров, заявок за неделю + выручка

Вход — `/admin/login` (email + пароль), доступ только пользователям с ролью `admin` в таблице `user_roles`.

## Что предлагаю добавить

Чтобы админка стала полноценной и удобной, добавлю следующее:

### 1. Товары — улучшения управления

- **Несколько фото за раз**: загрузка нескольких файлов одним кликом + drag&drop в зону загрузки.
- **Сортировка фото**: перетаскивание для смены порядка (первое фото — обложка). Сейчас порядок фиксированный по моменту загрузки.
- **Удаление файла из storage**: при удалении фото из карточки товара файл будет удаляться и из bucket `product-images`, чтобы не копился мусор.
- **Поиск и фильтры в каталоге товаров**: поиск по названию, фильтр по типу (Букет / Поштучно), цвету, статусу (активен / скрыт / нет в наличии).
- **Быстрое переключение «в наличии» и «активен»** прямо из карточки списка, без открытия редактора.
- **Дублирование товара**: кнопка «Создать копию» — удобно для серии похожих букетов.
- **Сортировка списка**: по дате, по названию, по цене.

### 2. Заказы — улучшения

- **Поиск** по имени клиента и телефону.
- **Фильтр по дате доставки** (сегодня / завтра / диапазон).
- **Реалтайм-обновление**: новые заявки появляются в списке без перезагрузки (через Supabase Realtime).
- **Звуковое/визуальное уведомление** при появлении новой заявки (бейдж в боковом меню с количеством новых).
- **Экспорт в CSV** отфильтрованного списка заявок.
- **Кликабельный телефон** (`tel:`) и кнопка «Скопировать адрес».

### 3. Дашборд — расширение

- График заявок по дням за последние 14 дней (recharts уже доступен).
- Топ-5 товаров по количеству продаж за месяц.
- Средний чек.
- Карточка «Сегодня к доставке» — заказы с `delivery_date = today`.

### 4. Backend

Никаких изменений схемы БД не требуется — все нужные поля уже есть.

Понадобятся только:
- Включение Realtime для таблицы `orders` (через миграцию `ALTER PUBLICATION supabase_realtime ADD TABLE public.orders`).
- Опционально — RLS-policy для удаления файлов из bucket `product-images` (для админов).

### 5. UX

- Подтверждение через красивый диалог (AlertDialog) вместо браузерного `confirm`.
- Skeleton-загрузка вместо пустых экранов.
- Сохранение черновика товара в localStorage — если случайно закрыли панель, при повторном открытии форма восстанавливается.

3. **Дашборд** — графики, топ-товаров, сегодня к доставке.
4. **Всё сразу** — реализую полным пакетом одной итерацией.
