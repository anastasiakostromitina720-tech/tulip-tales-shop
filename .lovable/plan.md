

## Генерация товаров для каталога

Создам набор демо-товаров для каждой категории (букеты по цветам + поштучно), вставлю их в базу через миграцию-сидинг с готовыми изображениями, чтобы они сразу были видны в каталоге и в админке для редактирования/удаления.

### Что будет добавлено

**Букеты (bouquet)** — по 2–3 товара на каждый цвет:
- Розовые (pink) — «Нежное утро», «Розовый рассвет»
- Белые (white) — «Белое облако», «Снежная классика»
- Жёлтые (yellow) — «Солнечный полдень», «Лимонад»
- Пионовидные (peony) — «Пионовидная роскошь», «Шёлк и пион»
- Микс (mix) — «Радужный микс», «Цветной фейерверк»

**Поштучно (by_stem)** — по 1–2 товара на цвет:
- Розовый, белый, жёлтый, микс — «Тюльпан розовый», «Тюльпан белый» и т. д. с `stems_count = 1` и ценой за штуку.

Итого ~14–16 товаров. У каждого: название, описание, состав, цена, цвет, изображения (1–3 шт.), `in_stock=true`, `active=true`.

### Изображения

Использую публичные фото тюльпанов с Unsplash (стабильные CDN-ссылки вида `https://images.unsplash.com/...`) — они уже поддерживаются функцией `resolveImage` (она пропускает абсолютные URL без изменений). Это позволит не загружать ничего в storage вручную и сразу увидеть карточки.

Для каждой категории подберу 2–3 релевантных фото нужного цвета.

### Реализация

Один шаг — SQL-миграция-сидинг:

```sql
INSERT INTO public.products (name, description, composition, price, type, color, stems_count, images, in_stock, active)
VALUES
  ('Нежное утро', '...', '25 розовых тюльпанов', 3500, 'bouquet', 'pink', 25, ARRAY['https://images.unsplash.com/...'], true, true),
  ...;
```

После выполнения товары появятся:
- На главной/в каталоге `/catalog` — отфильтрованные по типу и цвету.
- В админке `/admin/products` — карточки с кнопками «редактировать» и «удалить», которые уже работают через существующий компонент `AdminProducts`.

### Ничего не нужно менять в коде

Админка, фильтры каталога и карточки товаров уже работают с этими полями — только наполняем базу данными.
